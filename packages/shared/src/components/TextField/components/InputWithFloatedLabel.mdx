# InputWithFloatedLabel

<br/>
<br/>

## Overview
- 사용자가 텍스트를 입력해야 할 때 사용합니다. 예: 연락처, 결제 정보 입력.
- Material Design 3의 TextField 컴포넌트입니다.
- `filled`, `outlined` 두 가지 스타일을 지원하며, `outlined` prop이 없으면 기본적으로 `filled`가 적용됩니다.
- `value`, `onChange`, `label`, `type`은 필수 prop입니다.

## Usuage

```tsx
<InputWithFloatedLabel
    label="label"
    type={"number" | "text" | "email" | "password"}
    value=""
    onChange={() => {}}
    outlined
    type="text"
    disabled={false}
    prefix="$"
    suffix="@gmail.com"
    leadingIcon={<MdReactIcon />}
    trailingIcon={<MdReactIcon />}
    supportingText="supporting text"
    fixedHeight={3 | "undefined"}
    clear={false}
    error={""}
/>
```

## Properties

### Type: Filled, Outlined

**Outlined** 타입은 `filled`보다 시각적 강조가 낮습니다.  
폼처럼 여러 입력 필드가 함께 사용되는 경우, 복잡도를 줄이기 위해 적합합니다.

**다음 기준에 따라 선택하세요:**
- 앱의 전체 비주얼 스타일에 적합한가?
- UI의 목적에 부합하는가?
- 주변 컴포넌트(버튼 등)와 충분히 구분되는가?

> 단, **하나의 UI에 여러 타입을 혼합해서 사용하지 마세요.** 예: 한 폼에 `filled`, `outlined`을 섞어 사용 ❌
---

### Label
- 입력 내용이 무엇인지 명확하게 안내합니다.
- 항상 보여야 하며, 잘리거나 여러 줄로 표시되지 않아야 합니다.
- 필수 항목은 `*`로 표시하세요.
---

### fixedHeight
- 입력 필드를 한 줄 또는 여러 줄로 설정할 수 있습니다.
- 한 줄: `fixedHeight` 생략
- 여러 줄: 숫자를 넘겨 고정된 줄 수 지정 (예: `fixedHeight={3}`)
---

### prefix, suffix
- 텍스트 앞뒤에 부가 정보를 표시합니다.
- 예시:
  - prefix: 통화 기호 (`₩`, `$`)
  - suffix: 이메일 도메인 (`@gmail.com`), 단위 (`kg`, `cm`)
  ---

### supporting text
- 입력값에 대한 간단한 설명 또는 가이드 메시지입니다.
- `error`가 설정되면 `supportingText` 대신 에러 메시지가 표시됩니다.
---

### disabled
- 입력을 비활성화합니다.
---

### leadingIcon, trailingIcon
- 입력값의 의미를 보조하는 아이콘입니다.
- 입력 필드 왼쪽 또는 오른쪽에 표시됩니다.
---

### clear
- 입력값이 있을 때, 값 삭제 버튼을 표시합니다.
